## What to Build

### 1. examples/c/hello_hud.c (create)

A minimal C program demonstrating the retained-mode API:

1. Create context via `winpane_create()`
2. Create HUD with versioned config (`WINPANE_CONFIG_VERSION`, sizeof)
3. Add background rect (dark, rounded corners)
4. Add title text (white, large font)
5. Add value text (colored, smaller font)
6. Show surface
7. Event loop: `winpane_poll_event()` + `Sleep(16)`, print clicks
8. Cleanup: `winpane_surface_destroy()`, `winpane_destroy()`

Error checking on every call using `winpane_last_error()`. Include `<windows.h>` for `Sleep()`. Compile instructions in header comment.

Important notes for C examples:
- Use `event.event_type` (not `event.type` as in the proposal draft) since `type` is a C++ reserved keyword
- `winpane_poll_event` returns 0 when an event is available, 1 when no event, -1/-2 on error
- Add a comment near `winpane_last_error()` usage: the pointer is invalidated by the next winpane call on the same thread

### 2. examples/c/custom_draw.c (create)

Demonstrates the canvas/custom draw API:

1. Create context and HUD
2. Add retained-mode background rect
3. Show surface, brief sleep for window to appear
4. `winpane_surface_begin_draw()` to get canvas
5. Draw a bar chart using `fill_rounded_rect`, `draw_text`, `draw_line`
6. Add decorative elements (ellipse, stroke rect)
7. `winpane_surface_end_draw()` to flush
8. Print status, sleep loop
9. Cleanup

Similar to the Rust custom_draw example from Phase 1.

### 3. examples/c/CMakeLists.txt (create)

```cmake
cmake_minimum_required(VERSION 3.20)
project(winpane_examples C)

set(WINPANE_DIR "${CMAKE_SOURCE_DIR}/../../crates/winpane-ffi")
set(WINPANE_INCLUDE "${WINPANE_DIR}/include")
set(WINPANE_LIB_DIR "${CMAKE_SOURCE_DIR}/../../target/debug")

add_executable(hello_hud hello_hud.c)
target_include_directories(hello_hud PRIVATE ${WINPANE_INCLUDE})
target_link_directories(hello_hud PRIVATE ${WINPANE_LIB_DIR})
target_link_libraries(hello_hud winpane)

add_executable(custom_draw custom_draw.c)
target_include_directories(custom_draw PRIVATE ${WINPANE_INCLUDE})
target_link_directories(custom_draw PRIVATE ${WINPANE_LIB_DIR})
target_link_libraries(custom_draw winpane)
```

### 4. examples/c/build.bat (create)

Simple MSVC build script for consumers without CMake:

```bat
@echo off
set WINPANE=..\..\crates\winpane-ffi
set LIB_DIR=..\..\target\debug
cl /W4 /I %WINPANE%\include hello_hud.c /link /LIBPATH:%LIB_DIR% winpane.lib
cl /W4 /I %WINPANE%\include custom_draw.c /link /LIBPATH:%LIB_DIR% winpane.lib
```

### 7. CI header verification

**File:** `.github/workflows/ci.yml`

Add two steps after "Build":

1. The MSVC developer command prompt setup (current CI does NOT include this):
```yaml
      - uses: ilammy/msvc-dev-cmd@v1
```

2. The header verification:
```yaml
      - name: Verify C header
        run: |
          cl /c /W4 /I crates/winpane-ffi/include crates/winpane-ffi/include/winpane.h
        shell: cmd
```

The `ilammy/msvc-dev-cmd@v1` action puts `cl.exe` on PATH. Without it, the `cl` command will fail on GitHub `windows-latest` runners.

1. `cargo build --workspace` passes, producing `winpane.dll` + `winpane.lib`

3. `cargo test --workspace` passes (existing P1/P2 tests still pass)
4. `include/winpane.h` is auto-generated and compiles without warnings
5. `examples/c/hello_hud.c` and `custom_draw.c` compile and link against `winpane.lib`
6. `examples/rust/custom_draw.rs` compiles
7. All 35 symbols present in DLL exports
8. `cargo fmt --all -- --check` passes
