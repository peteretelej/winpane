# Phase 2: FFI Crate Setup and Error Handling

## Overview

Transform the `winpane-ffi` crate from a stub into a buildable cdylib with cbindgen header generation and the error handling infrastructure that all FFI functions depend on. After this phase, the crate compiles and exports `winpane_last_error()` as its first real symbol.

## Prerequisites

- Phase 1 complete (DrawOp and custom_draw exist in the workspace)
- Read `initial-plan.md` Phases 3-4
- Read `proposal.md` "Error handling" section
- Current files: `crates/winpane-ffi/Cargo.toml` (stub), `crates/winpane-ffi/src/lib.rs` (stub)

## What to Build

### 1. Update Cargo.toml

**File:** `crates/winpane-ffi/Cargo.toml`

Replace entire file:

```toml
[package]
name = "winpane-ffi"
version = "0.0.1"
edition.workspace = true
license.workspace = true
repository.workspace = true
description = "C ABI bindings for winpane - Windows companion surface SDK"

[lints]
workspace = true

[lib]
name = "winpane"
crate-type = ["cdylib", "staticlib"]

[dependencies]
winpane = { version = "0.0.1", path = "../winpane" }

[build-dependencies]
cbindgen = "0.29"
```

Note: `name = "winpane"` forces the output to `winpane.dll` + `winpane.lib` instead of `winpane_ffi.dll`. This matches the proposal and .def file expectations.

Notes:
- `cdylib` produces `winpane.dll` + `winpane.lib` on Windows
- `staticlib` included for consumers who want static linking
- No `libc` dependency needed; use `std::os::raw::{c_char, c_int}` from std
- `cbindgen = "0.29"` for header generation (latest stable as of Feb 2026)

### 3. Create build.rs

**File:** `crates/winpane-ffi/build.rs` (create)

```rust
fn main() {
    let crate_dir = std::env::var("CARGO_MANIFEST_DIR").unwrap();

    let config = cbindgen::Config::from_file(format!("{crate_dir}/cbindgen.toml"))
        .expect("failed to read cbindgen.toml");

    cbindgen::Builder::new()
        .with_crate(crate_dir)
        .with_config(config)
        .generate()
        .expect("cbindgen failed to generate header")
        .write_to_file("include/winpane.h");
}
```

Create the output directory: `crates/winpane-ffi/include/` (mkdir).

**cbindgen parse note:** cbindgen may fail if it can't parse the crate. During initial development, consider wrapping the generate call in `if let Ok(...)` with a warning, then making it strict before the final phase. The initial plan notes this mitigation.

### 4. Error handling infrastructure in lib.rs

**File:** `crates/winpane-ffi/src/lib.rs`

Replace the 2-line stub with the error handling foundation:

**Module-level attributes and imports:**
```rust
#![allow(clippy::missing_safety_doc)]

use std::cell::RefCell;
use std::ffi::{CStr, CString};
use std::fmt;
use std::os::raw::c_char;
use std::panic::AssertUnwindSafe;
```

**Thread-local error storage:**
```rust
thread_local! {
    static LAST_ERROR: RefCell<Option<CString>> = const { RefCell::new(None) };
}

fn set_last_error(msg: impl fmt::Display) { ... }
```

**`winpane_last_error()` function:**
- `#[no_mangle] pub extern "C" fn`
- Returns `*const c_char` - the last error message, or NULL if no error
- Pointer valid until the next winpane call on the same thread

**`ffi_try!` macro:**
- Wraps function bodies in `catch_unwind(AssertUnwindSafe(|| ...))`
- Returns `0_i32` on `Ok(())`
- Returns `-1_i32` on `Err(e)` with `set_last_error(&e)`
- Returns `-2_i32` on panic with `set_last_error("panic caught at FFI boundary")`

**`ffi_try_with!` macro** (variant for out-pointer functions):
- Same pattern but returns the Ok value instead of 0
- Uses `return -1_i32` / `return -2_i32` for error paths

**Helper functions:**
- `require_non_null<T>(ptr: *const T, name: &str) -> Result<(), String>` - validates non-null
- `require_non_null_mut<T>(ptr: *mut T, name: &str) -> Result<(), String>` - validates non-null mut
- `unsafe fn cstr_to_string(ptr: *const c_char) -> Result<String, String>` - null check + CStr::from_ptr + to_str + to_owned

See `initial-plan.md` Phase 4 for the full implementation code.
